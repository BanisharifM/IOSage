Why Polaris Dataset Enables Superior Bottleneck Detection

The ALCF Polaris collection offers significant advantages over the tabular AIIO CSV format for comprehensive I/O performance bottleneck detection.

1. Multi-Module Coverage

Bottleneck Type AIIO Polaris
POSIX-level inefficiencies Partial Full
MPI-IO collective issues None Full (65+ counters)
STDIO buffering problems None Full (30+ counters)
Metadata overhead Partial Full (F_META_TIME)
Temporal patterns None Full (HEATMAP)
Rank imbalance None Full (VARIANCE_RANK_*)

2. Critical Timing Information

AIIO Dataset - No Timing

Only operation counts and sizes. Cannot distinguish between:

  • 1000 reads completed quickly (efficient)
  • 1000 reads completed slowly (bottleneck)

Polaris Dataset - Full Timing

  • F_READ_TIME, F_WRITE_TIME
  • F_META_TIME - metadata overhead
  • F_FASTEST_RANK_TIME, F_SLOWEST_RANK_TIME
  • F_VARIANCE_RANK_TIME - load imbalance
  • Full timestamps for temporal analysis

3. Rank-Level Statistics (Parallel I/O Analysis)

Polaris-Exclusive Counters for Load Imbalance Detection

FASTEST/SLOWEST_RANK

Identify straggler ranks

VARIANCE_RANK_BYTES

Data distribution imbalance

VARIANCE_RANK_TIME

Timing consistency

4. Complete Size Histograms

Size bucket coverage comparison:

AIIO Buckets
5 ranges
Polaris Buckets
10 ranges (complete)

Missing in AIIO: 10K-100K, 1M-4M, 4M-10M, 10M-100M, 100M-1G, 1G+ buckets

5. Bottleneck Detection Capabilities

Bottleneck Category AIIO Polaris Key Enabling Counters
Small I/O operations Partial Full Complete SIZE_* histograms
Non-sequential access Yes Yes SEQ_*, CONSEC_*
Poor alignment Yes Yes *_NOT_ALIGNED
Read/write imbalance Partial Full Timing + bytes combined
Metadata overhead Limited Full F_META_TIME, STATS
Collective I/O misuse No Full MPIIO_COLL_*, MPIIO_INDEP_*
Load imbalance No Full *_RANK statistics
Temporal patterns No Full HEATMAP, timestamps
File contention No Yes Per-file records

6. ML-Based Detection Advantages

1
Multi-label Classification

Detect multiple simultaneous bottlenecks with rich feature set

2
Severity Estimation

Use timing data to quantify bottleneck impact

3
Root Cause Analysis

Trace issues to specific files and ranks

4
Self-Supervised Pre-training

Pre-train on 1.37M unlabeled logs, fine-tune on benchmarks

Data Conversion Commands

# Full text dump with all options darshan-parser --all file.darshan > output_full.txt # Performance metrics only darshan-parser --perf file.darshan > output_perf.txt # Aggregated totals darshan-parser --total file.darshan > output_total.txt # Python extraction with PyDarshan python3 -c " import darshan report = darshan.DarshanReport('file.darshan') print(report.metadata['job']) "
